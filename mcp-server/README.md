# Operator MCP Server

MCP server bridging Claude Code sessions to the Operator daemon. Enables Claude Code sessions to register with Operator, speak through Operator's audio queue, and push context updates.

## Prerequisites

- Node.js (v20+)
- The Operator daemon must be running on `localhost:7420`
- A valid auth token at `~/.operator/token` (generated by the Operator daemon on first launch)

## Build

```bash
cd mcp-server
npm install
npx tsc
```

The compiled output is written to `build/index.js`.

## Installation

Copy the built files to the deployment location:

```bash
sudo mkdir -p /usr/local/lib/operator/mcp-server
sudo cp -r build package.json package-lock.json node_modules /usr/local/lib/operator/mcp-server/
```

Add the MCP server entry to your global Claude Code MCP configuration at `~/.claude/mcp.json`:

```json
{
  "mcpServers": {
    "operator": {
      "command": "node",
      "args": ["/usr/local/lib/operator/mcp-server/build/index.js"]
    }
  }
}
```

If `~/.claude/mcp.json` already exists with other servers, merge the `operator` entry into the existing `mcpServers` object.

## Auto-Registration

Claude Code sessions auto-register with Operator on startup. The project-level `CLAUDE.md` instructs each session to call `register()` with:

- **name**: derived from the working directory basename (e.g., `frontend`, `api`)
- **tty**: the TTY path obtained by running `tty` in bash (e.g., `/dev/ttys004`)

This happens automatically when the operator MCP server is available to the Claude Code session.

## MCP Tools

| Tool | Description |
|------|-------------|
| `register` | Register this Claude Code session with Operator. Call on startup with session name, TTY path, and optional context. |
| `speak` | Speak a message to the user through Operator's audio queue. Use instead of the `say` command. |
| `update_context` | Update session context for routing. Call after significant work changes so Operator can route messages accurately. |
| `get_state` | Check Operator state including whether the user is speaking, queue status, and active sessions. |
