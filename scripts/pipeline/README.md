# Qwen3.5 Cleanup Pipeline

On-device speech-to-text cleanup using Qwen3.5-0.8B on CoreML.

## Architecture

```
Pre-computed cache (131 tokens, 13MB FP16, loads in 16ms)
        ↓
Batch Prefill (chunked DeltaNet, blocked forward substitution)
  → 64 tokens in 39ms, single CoreML predict()
        ↓
Per-token Decode (monolith KV cache model)
  → 83 tok/s, 12ms/step
```

## Warm Latency (measured via CleanupCLI)

| Phase | Time |
|---|---|
| Tokenize (Python) | ~1ms |
| Batch prefill | ~39ms |
| No-think skip | ~23ms |
| Generate (~10 tok) | ~120ms |
| Detokenize | ~1ms |
| **Total** | **~180ms** |

## Files

| File | Purpose |
|---|---|
| `convert_decode.py` | Trace + convert decode model (monolith KV FP16) |
| `convert_prefill.py` | Trace + convert batch prefill model (chunked DeltaNet FP16) |
| `gen_prompt_cache.py` | Pre-compute system prompt states |
| `tokenizer_helper.py` | Runtime Python tokenizer (stdin/stdout IPC) |
| `patches.py` | coremltools op fixes (view, reshape, pad, slice, etc.) |
| `functional_deltanet.py` | HF model patches (per-token recurrence, KV cache) |
| `functional_deltanet_chunked.py` | Chunked DeltaNet kernel (blocked forward substitution) |
| `solve_triangular_blocked.py` | Blocked (I-L)⁻¹ solve — FP16-safe, 8×8 blocks |
| `verify_prefill.py` | Correctness verification vs PyTorch reference |

## Regenerate from Scratch

```bash
PYTHON=/tmp/coreml-venv/bin/python

# 1. Decode model (monolith, FP16, ~30s)
$PYTHON scripts/pipeline/convert_decode.py Qwen/Qwen3.5-0.8B \
    --output models/monolith_kv_fp16/

# 2. Pre-compute system prompt cache (~5s)
$PYTHON scripts/pipeline/gen_prompt_cache.py \
    --output models/monolith_kv_fp16/prompt_cache/

# 3. Batch prefill model (chunked DeltaNet, FP16, ~80s)
$PYTHON scripts/pipeline/convert_prefill.py Qwen/Qwen3.5-0.8B \
    --output models/batch_prefill_fp16/

# 4. Download tokenizer files
mkdir -p models/monolith_kv_fp16/tokenizer
curl -sL https://huggingface.co/Qwen/Qwen3.5-0.8B/resolve/main/tokenizer.json \
    -o models/monolith_kv_fp16/tokenizer/tokenizer.json
curl -sL https://huggingface.co/Qwen/Qwen3.5-0.8B/resolve/main/tokenizer_config.json \
    -o models/monolith_kv_fp16/tokenizer/tokenizer_config.json

# 5. Test
echo "um so I was like going to the uh store" | swift run -c release CleanupCLI
```

## Key Design Decisions

- **Blocked forward substitution** over Neumann product: keeps matrix powers ≤ L⁴
  within 8×8 blocks, preventing FP16 overflow in state matmuls (state values ~8000,
  128-dim sums would hit ~1M with Neumann's L³²)
- **Two separate models** (prefill + decode): same weight packing order, states hand
  off directly via matching `conv_state_N`/`rec_state_N`/`key_cache_N` names
- **Conv state slicing**: prefill outputs `[1, 6144, 68]`, decode expects `[1, 6144, 4]` —
  take last 4 elements at handoff
- **Pre-computed cache in FP16**: generated by PyTorch FP32, saved as FP16 .npy.
  Apple Silicon GPU applies mixed precision to FP32 models anyway
